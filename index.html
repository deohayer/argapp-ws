API

This chapter describes entities available to the developer and has a specific structure:

  • Each subchapter represents a scope: packages, modules, classes etc. The scope only has a description and an arbitrary number of sections.

  • Each section represents either a variable or a function.

    • The variable refers to variables, constants, fields, properties, etc. It has a description with the declaration, and the following optional subsections:

      • Examples - usage examples.

      • Exceptions - exceptions when the variable is set to some invalid value.

      • Defaults - evaluation order if the variable is not set explicitly.

    • The function refers to functions, methods, decorators, etc. It has a description with the declaration, and the following optional subsections:

      • Examples - usage examples.

      • Parameters - a list of parameters accepted by the function.

      • Exceptions - exceptions when invalid parameter values are provided.

      • Returns - a list of possible return values.

argapp

The package is a module. It exports the following identifiers:

  • Arg - a class that represents a command line argument.

  • App - a class that represents a command.

  • CallError - an exception to raise when there is an error in App.__call__().

  • main() - the main function: parses the command line and executes the commands.

argapp.main(App, list[str])

def main(
    app: App,
    argv: list[str] = sys.argv,
) -> None:
    ...

This is the main function of the script. It does the following:

  1. Parses argv and generates parameters args and apps for App.__call__().

  2. For each app in apps calls app(args, apps).

  3. Any exception is catched and printed.

  4. sys.exit() is called with an appropriate exit code.

Parameters

  • app - the App instance to use as the main command.

  • argv - the command line to parse, as a list of individual tokens: ['git', 'push', '--force', 'origin', 'main'].

    • Defaults to sys.argv.

    • os.path.basename(argv[0]) is used as the command name in the help text in case if app.name is None.

Exceptions

  • SystemExit, if any error is raised:

    • If the type of error is CallError, it is a user-friendly error:

      • The usage text is printed to sys.stderr, if the error originates from the parsing.

      • error.message is printed to sys.stderr, if not None.

      • SystemExit(error.code) is raised.

    • If the type of error is not CallError, it is a developer-friendly error:

      • str(error) with its traceback is printed to sys.stderr.

      • SystemExit(1) is raised.

    • SystemExit(0) is raised.

argapp.Arg

A class which represents an optional or a positional argument.

  • Positional arguments:

    • Have unique value names within the same App.

    • Always required. Setting count to '*' and '?' mimics the optional behavior.

    • Appear in the command line in a strict order.

  • Optional arguments:

    • Have unique short and long option names within the same App. The value name is not necessarily unique.

    • Can be omitted completely unless marked as required.

    • Can appear in the command line in any order and be mixed with the positional. Providing optional arguments before positional is encouraged.

Important concepts:

  • The same Arg can be associated with multiple App.

  • The value of Arg is retrieved using the instance itself. It is usually necessary to store the constructed Arg.

  • All fields are properties. Note that "get" and "set" types do not match.

  • Setting the field to None means that it will be implicitly re-evaluated according to "Defaults".

Arg.count

class Arg:
    ...
    count: int | str = ...

A number of values consumed from the command line by the argument. Can be one of:

  • 0 (only if self.optional is True).

  • A positive integer value.

  • '?', 0 or 1 values.

  • '*', 0 or more values.

  • '+', 1 or more values.

  • '~', the rest of the command line (only if self.positional is True).

Defaults

  • '*', if the type of self.default) is list.

  • 1.

Arg.default

class Arg:
    ...
    default: object | list | None = ...

A default value to be used if the actual value was not provided. It is returned by the base implementations of Arg.__call__() in the following cases:

  • If self.count is '?' or '*', and zero values are provided in the command line.

  • If self.suppress is False, and the argument is not mentioned in the command line.

Defaults

  • False, if self.flag is True.

  • [], if self.count is '*'.

  • None.

Any Iterable can be used to initialize the value, it will be converted to dict, setting the dictionary values to None.

Defaults

argapp.App

A class that represents a command or a subcommand. It is supposed to be subclassed to handle the parsed command line in App.__call__().

Important concepts:

  • The same App can be associated with multiple App.

  • All fields are internally properties:

    • "get" types can be seen in the field`s type hints for all fields.

    • "set" types can be seen in the type hints of App.__init__(**) for the writeable fields.

  • Setting the field to None means that it will be implicitly re-evaluated according to "Defaults".

App.arg_help

class App:
    ...
    arg_help: Arg = ...

The argument for the help text:

  • It is not added to self.args, not used as is, and serves only as a data provider.

  • Only the sopt, lopt and help are used, the rest is ignored.

  • If sopt and lopt are None, the help text option is disabled.

Defaults

  • Arg(sopt='h', lopt='help', help='Show the help text and exit.').

  • Only the name and help are used, the rest is ignored.

  • If name is None, the help text option is disabled.

Defaults

  • Arg(name='CMD', help='A subcommand to run.').

argapp.CallError

A subclass of RuntimeError. These exceptions are supposed to be raised in App.__call__() with custom return code and error message in case of any errors during the execution.

The fields are checked upon setting. None is always a valid value which means that the field has to use its defaults.
